# サーベイ構成案の最終化

## 入力

- `$ARGUMENTS`: 構成に関する追加指示（省略可）
  - 例: `/outline 時系列ではなく手法カテゴリ別に整理してほしい`

## 前提

- `.claude/context/scope.md` が存在すること
- `.claude/context/notes.md` が存在すること（Paper Card 形式）
- `.claude/context/draft_outline.md` が存在する場合はそれをベースにする（/read で反復更新済み）

## 研究分野情報

- **ドメイン**: {{DOMAIN}}

## プロンプト

draft_outline.md（存在する場合）または notes.md の Paper Card 群をもとに、サーベイ論文の最終構成案を作成してください。

### 構成案に含める内容

1. **サーベイタイトル案**（2〜3候補）

2. **セクション構成** — 各セクションについて:
   - セクション番号・タイトル
   - 目的（このセクションで何を伝えるか）
   - 対応する RQ
   - 含める論文（notes.md の BibTeX Key で参照）
   - 想定ページ数（目安）

3. **論文のセクション割り当てマトリクス**:
   | BibTeX Key | Sec.2 | Sec.3 | Sec.4 | ... |
   |------------|-------|-------|-------|-----|
   | he2016deep | ★    |       | ○    |     |
   （★ = 主要引用、○ = 補足引用）

4. **RQ カバレッジ確認**:
   - 各 RQ がどのセクションで回答されるか
   - カバーされていない RQ があれば警告

5. **ギャップ分析**:
   - draft_outline.md のギャップメモを反映
   - カバーが薄いトピック
   - 追加検索が必要な領域 → `/search` を提案

### 推奨するサーベイ構成

```
1. Introduction — 背景、動機、RQ、サーベイの貢献
2. Background — 前提知識、用語定義
3-N. 本論セクション — テーマ別 or 手法別 or 時系列
N+1. Discussion — 横断的分析、トレンド、課題
N+2. Conclusion — RQ への回答まとめ、今後の方向性
```

## ルール

- scope.md の RQ を全てカバーする構成にすること
- notes.md に含まれない論文を参照しない（BibTeX Key で照合）
- セクション間の論理的なつながりを明示する
- 1セクションに論文を詰め込みすぎない（目安: 1セクション 5〜10 論文）
