# 研究スコープの定義

## 入力

- `$ARGUMENTS`: 研究トピックの説明（必須）
  - 例: `/scope Vision Transformer のサーベイ — 画像認識タスクにおける ViT 系手法の発展と課題`

## 研究分野情報

- **ドメイン**: {{DOMAIN}}
- **主要会議・ジャーナル**: {{KEY_VENUES}}

## プロンプト

`$ARGUMENTS` をもとに、以下を含む研究スコープ文書を作成してください。

### 1. Research Questions (RQs)

3〜5個の明確な研究課題を定義する。各 RQ は:
- 具体的で回答可能な問い
- サーベイ全体のストーリーラインを構成する順序

### 2. 検索キーワード

- 主キーワード（`$ARGUMENTS` から抽出）
- 関連キーワード（以下のドメインキーワードから関連するものを選択＋追加）:
  {{DOMAIN_KEYWORDS}}
- 検索クエリ例（キーワードの組み合わせ）を3〜5パターン

### 3. 包含・除外基準

| 基準 | 包含 | 除外 |
|------|------|------|
| 年代 | （トピックに応じて設定） | |
| 言語 | 英語 | |
| 文献種別 | 査読付き論文、プレプリント | 学位論文、テクニカルレポート |
| 会議/ジャーナル | {{KEY_VENUES}} + 関連 | |
| 手法/対象 | （トピックに応じて設定） | |

### 4. 対象会議・ジャーナル

`{{KEY_VENUES}}` から選択し、トピックに応じて追加。

### 5. 期待するサーベイの構成（初期案）

大まかなセクション構成を提案する（後で /outline で詳細化）。

## ルール

- RQ はユーザーの `$ARGUMENTS` から逸脱しないこと
- 包含/除外基準は厳密すぎず、初期検索で十分な論文が見つかる範囲にする
- 不明確な点があれば AskUserQuestion で確認（最大3問）

## 出力

`.claude/context/scope.md`

## 次のステップ

> scope.md を確認してください。

> 問題なければ `/search` で文献検索に進んでください。
